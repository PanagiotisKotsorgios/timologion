  Θέμα: Συνεργασία με timologion — SaaS πλατφόρμα ηλεκτρονικής τιμολόγησης

  Αγαπητή ομάδα της Wrapp,

  Ονομάζομαι Παναγιώτης Κοτσόργιος και είμαι ο δημιουργός του
  timologion (https://github.com/PanagiotisKotsorgios/timologion) —
  μιας νέας ελληνικής SaaS πλατφόρμας ηλεκτρονικής τιμολόγησης που
  αναπτύσσεται με στόχο να απευθυνθεί σε ελεύθερους επαγγελματίες
  και μικρομεσαίες επιχειρήσεις.

  ────────────────────────────────────────────────────────────────
  ΤΙ ΕΙΝΑΙ ΤΟ TIMOLOGION
  ────────────────────────────────────────────────────────────────

  Το timologion είναι μια σύγχρονη web εφαρμογή (Next.js + Prisma +
  MySQL) που δίνει στους χρήστες:

  • Καθαρό dashboard για διαχείριση πελατολογίου, ειδών, υπηρεσιών
    και παραστατικών.
  • Έκδοση όλων των τύπων παραστατικών (τιμολόγια πώλησης,
    παροχής υπηρεσιών, αποδείξεις λιανικής, πιστωτικά, προτιμολόγια,
    προσφορές, παραγγελίες, δελτία αποστολής).
  • Επαναλαμβανόμενα παραστατικά με αυτόματο scheduler.
  • Πληρωμές, μερικές εξοφλήσεις, ανεξόφλητα υπόλοιπα.
  • POS module με τραπέζια εστιατορίου, γρήγορη πώληση και
    θερμικές αποδείξεις 80mm.
  • CRM με leads, ευκαιρίες σε kanban pipeline, εργασίες
    follow-up και υπενθυμίσεις.
  • Εξαγωγή σε CSV / εισαγωγή προϊόντων από CSV.
  • Απόθεμα, ζώνες τιμών (χονδρική, λιανική, VIP).
  • Αναφορές ΦΠΑ ανά συντελεστή και εσόδων-εξόδων για τον
    λογιστή.
  • Πολλαπλές επιχειρήσεις ανά χρήστη, ρόλοι πρόσβασης
    (owner / admin / λογιστής / πωλητής / staff / readonly),
    audit log για κρίσιμες ενέργειες.
  • Πλήρες mobile-friendly interface.
  • Σύνδεση με Google και Facebook (OAuth), 2FA με TOTP,
    επιβεβαίωση email.
  • Ασφάλεια: Argon2id για κωδικούς, AES-256-GCM για ευαίσθητα
    δεδομένα, rate limiting, security headers, cookie banner
    συμμορφωμένο με GDPR.

  Η πλατφόρμα έχει σχεδιαστεί από την αρχή έτσι ώστε η
  φοροσήμανση, η ανάθεση MARK/UID/QR και η διαβίβαση στο myDATA
  να γίνονται αποκλειστικά μέσω πιστοποιημένου παρόχου
  ηλεκτρονικής τιμολόγησης — και θα θέλαμε αυτός ο πάροχος να
  είναι η Wrapp.

  Σε όλα τα δημόσια σημεία της πλατφόρμας (marketing site, όροι
  χρήσης, πολιτική απορρήτου, footer, onboarding) αναφέρεται ρητά
  ότι η Wrapp είναι ο συνεργαζόμενος πιστοποιημένος πάροχος
  ΥΠΑΗΕΣ και ότι εμείς δεν αποτελούμε πάροχο ηλεκτρονικής
  τιμολόγησης — παρέχουμε τη user-friendly εμπειρία, το dashboard
  και την επιχειρηματική οργάνωση.

  ────────────────────────────────────────────────────────────────
  ΤΟ ΑΙΤΗΜΑ ΜΑΣ
  ────────────────────────────────────────────────────────────────

  Είμαστε έτοιμοι να ενσωματώσουμε την Wrapp API στην πλατφόρμα
  μας και να ξεκινήσουμε δοκιμές σε staging περιβάλλον. Πριν
  προχωρήσουμε, θα θέλαμε να συζητήσουμε δύο βασικά σημεία:

  1) ΕΙΔΙΚΕΣ ΠΡΟΣΦΟΡΕΣ ΓΙΑ ΤΟΥΣ ΧΡΗΣΤΕΣ ΜΑΣ

      Επειδή το timologion θα αποτελέσει το κύριο σημείο εισόδου
      πολλών ελληνικών επιχειρήσεων στην ηλεκτρονική τιμολόγηση,
      θα θέλαμε να προσφέρουμε στους χρήστες μας ειδικές τιμές
      ή pricing brackets μέσω της συνεργασίας μας.

      Ενδιαφερόμαστε είτε για:
      • Ειδικό partner rate που μπορούμε να μεταφέρουμε στους
        χρήστες, ή
      • Επιπλέον έκπτωση όγκου (volume discount) καθώς αυξάνεται
        η βάση χρηστών μας, ή
      • Rebate / revenue share μοντέλο ανά μεταβίβαση.

  2) ΜΗΝΙΑΙΑ ΠΛΑΝΑ (ΟΧΙ ΜΟΝΟ ΕΤΗΣΙΑ) ΓΙΑ ΟΛΟΥΣ ΤΟΥΣ ΧΡΗΣΤΕΣ

      Το βασικό feedback που έχουμε πάρει από τους ελεύθερους
      επαγγελματίες και τις μικρές επιχειρήσεις είναι ότι
      προτιμούν μηνιαία χρέωση αντί για εφάπαξ ετήσια — για
      λόγους cash flow και ευελιξίας.

      Θα θέλαμε να είναι διαθέσιμα μηνιαία πλάνα για ΟΛΟΥΣ τους
      χρήστες μας, και όχι μόνο ετήσια, ώστε ο κάθε νέος χρήστης
      να μπορεί να ξεκινήσει με χαμηλότερη αρχική δέσμευση.

      Αν αυτή τη στιγμή δεν διατίθεται μηνιαίο πλάνο για το
      partner tier που θα εντάξεις τους χρήστες μας, μπορούμε να
      συζητήσουμε πώς θα το ενεργοποιήσουμε συγκεκριμένα για την
      πελατειακή μας βάση;

  ────────────────────────────────────────────────────────────────
  ΕΠΟΜΕΝΑ ΒΗΜΑΤΑ
  ────────────────────────────────────────────────────────────────

  • Θα χαρούμε πολύ να κανονίσουμε ένα σύντομο call (15–30
    λεπτά) για να παρουσιάσουμε το προϊόν, να δείξουμε το UI και
    να συζητήσουμε τους όρους συνεργασίας.
  • Είμαστε έτοιμοι να ξεκινήσουμε δοκιμαστική ενσωμάτωση στο
    staging περιβάλλον σας — χρειαζόμαστε staging API keys και
    documentation access για το partner endpoint.
  • Έχουμε ήδη υλοποιήσει τη δομή για ανταλλαγή δεδομένων
    παραστατικών (client factory + Zod-validated DTOs) οπότε
    αναμένουμε γρήγορη ολοκλήρωση του integration μόλις
    ξεκινήσουμε.
